# Agent Guide for Taskboard Development

This document provides guidelines for AI agents (such as Gemini CLI, Cursor, or GitHub Copilot) contributing to the development and maintenance of the Taskboard application itself.

## Language Policy
- **Code & Documentation**: All source code, comments, and documentation (including this file) must be written in **English**.
- **Communication**: Interaction with the user via chat is primarily in **Japanese**, unless otherwise requested.

## Project Philosophy
- **Zero Dependencies**: Built using standard Web technologies (Vanilla JS/HTML/CSS). Avoid adding external libraries (React, Tailwind, etc.).
- **Local First**: No server-side logic. All data operations are performed directly on the local file system via the **File System Access API**.
- **AI-Friendly Data**: Persists data in **NDJSON** (Newline Delimited JSON) format to be easily parsed and generated by both humans and AI tools.

## Technical Stack
- **HTML (`taskboard.html`)**: Defines the UI structure and component templates using `<template>` tags.
- **CSS (`taskboard.css`)**: Manages styles and themes (Light/Dark) using CSS Custom Properties.
- **JavaScript (`taskboard.js`)**: Contains the core logic for file I/O, state management, and DOM manipulation.
- **Server (`start.py`)**: A simple Python script to serve the application locally for development.

## Development Mandates
1. **DOM Manipulation**: 
   - Centralize element references in the `UI` object within `taskboard.js`.
   - Use `<template>` tags for generating dynamic elements (cards, columns). Do not construct complex HTML strings manually.
2. **Data Integrity & Logging**:
   - `tasks.ndjson` is the source of truth for the current state (snapshot).
   - `events.ndjson` is an append-only log for all mutations.
   - Always call `appendEvent()` when implementing new task-related actions.
3. **Compatibility**: Target the latest versions of Chrome and Edge (File System Access API support required).
4. **Coding Style**:
   - Use `UPPER_SNAKE_CASE` for constants and `camelCase` for functions/variables.
   - Keep comments concise and explain the "why" behind complex logic.

## Validation & Testing
1. Run `python start.py` and navigate to `http://localhost:8000`.
2. Use the `sample-project/` directory to verify changes against existing data.
3. Verify UI changes in both Light and Dark themes.

## Directory Structure Notes
- `sample-project/`: For manual testing only. Its `.taskboard` directory is ignored by Git.
- `.taskboard/README.md`: Contains project-specific AI instructions and data format details. This is generated for each project and is separate from this developer guide.
